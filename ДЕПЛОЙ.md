# Бот на Render (24/7 без Mac)

## Что нужно (один раз)

| Что | Зачем |
|-----|--------|
| **GitHub** | Хранить код |
| **GitHub Token (classic, ghp_)** | Загрузить код с Mac |
| **Render** | Сервер в облаке — [render.com](https://render.com) |
| **Секреты из `.env`** | Токен бота, Polza, ЮKassa |

---

## Запуск

```bash
cd interior-telegram-bot
chmod +x deploy-online.sh check-github-token.sh
./deploy-online.sh
```

1. Создайте пустой репозиторий на GitHub: `interior-telegram-bot` (без README)
2. Скрипт загрузит код
3. На Render: **Blueprints** → **New Blueprint Instance** → выберите репозиторий → **Apply**
4. При первом деплое введите секреты:
   - `TELEGRAM_BOT_TOKEN`
   - `POLZA_API_KEY`
   - `YOOKASSA_SHOP_ID`
   - `YOOKASSA_SECRET_KEY`

---

## Остановить бота на Mac

Два процесса с одним токеном Telegram — конфликт. На Mac:

```bash
./service.sh stop
```

---

## ЮKassa webhook (автоактивация после оплаты)

В личном кабинете ЮKassa → HTTP-уведомления:

```
https://interior-telegram-bot.onrender.com/webhook/yookassa
```

Кнопка «Проверить оплату» работает и без webhook.

---

## Бесплатный тариф Render

- Сервис может **засыпать** после ~15 мин без HTTP-запросов.
- Чтобы бот не засыпал: бесплатный [UptimeRobot](https://uptimerobot.com) — пинг каждые 5 мин:
  `https://interior-telegram-bot.onrender.com/health`
- Подписки на **бесплатном** тарифе хранятся в контейнере — сбрасываются при пересборке. Для постоянной БД — платный тариф + диск в Render.

## Обновление кода

```bash
./deploy-online.sh
```

Render пересоберёт образ автоматически после push в `main`.

---

## Автодеплой (после Blueprint на Render)

1. API-ключ: https://dashboard.render.com/u/settings#api-keys → **Create API Key**
2. В `.env` добавьте: `RENDER_API_KEY=rnd_...`
3. Запустите:

```bash
./render-sync.sh
```

Скрипт подставит секреты из `.env` на Render, запустит деплой и остановит бота на Mac.

Полный цикл (GitHub + Render):

```bash
export GITHUB_TOKEN=ghp_...
export RENDER_API_KEY=rnd_...
./render-deploy.sh
```
